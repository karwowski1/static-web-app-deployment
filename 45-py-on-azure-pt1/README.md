# Containerized App on Azure (Part 3)

This project provisions a containerized Python application on Azure Container Apps, secured behind an Azure Application Gateway, with an integrated CI/CD pipeline using Jenkins.

## Architecture & Features
- **Azure Container Apps (ACA)**: Hosts the Python application.
- **Auto-Scaling**: Configured via KEDA CPU scaler. The app scales out when average CPU utilization exceeds 50% (Max 3 replicas) and scales in automatically during low utilization.
- **Azure Application Gateway**: Acts as a Layer 7 Load Balancer, providing public access and protecting the backend Container App.
- **Log Analytics**: Centralized logging for the Application Gateway and Container Apps, with data retention configured for 30 days.
- **Azure Container Registry (ACR)**: Private registry storing the application's Docker images.

## Continuous Integration & Continuous Deployment (CI/CD)
The CI/CD process is managed via Jenkins using the `Jenkinsfile` at the root of the repository.

### Pipeline Stages
1. **Build & Push**: Builds the Docker image from the `./app` directory and pushes it to Azure Container Registry.
2. **Terraform Init & Validate**: Initializes Terraform, formats code (`terraform fmt`), runs validation, and checks for best practices (`tflint`).
3. **Terraform Plan**: Generates an execution plan using a Service Principal scoped *only* for planning (Read-only access). The plan is saved as a pipeline artifact (`tfplan`) to prevent configuration drift. This stage runs automatically when code is pushed.
4. **Terraform Apply**: Applies the previously generated `tfplan` using a Service Principal scoped with Contributor access. **This stage requires manual approval** in the Jenkins UI.

### Concurrency
The pipeline utilizes the Jenkins `lock('terraform-state')` block to prevent multiple jobs from applying changes to the remote state simultaneously.

### Pipeline Authentication
The Jenkins pipeline uses OIDC (Workload Identity Federation) via the Azure Credentials Plugin to assume two distinct Azure AD Service Principals dynamically, avoiding the need to store long-lived secrets in Jenkins.

## Operational Guide

### How to View Error Pages
Error pages (e.g., 502 Bad Gateway or 404 Not Found) generated by the Application Gateway or Container App can be tracked in the Azure Portal:
1. Navigate to your Application Gateway or Container App resource.
2. Go to the "Metrics" or "Insights" blade to view HTTP response codes.

### How to Access Logs
Logs are retained for 30 days in the configured Log Analytics Workspace.
1. In the Azure Portal, navigate to your **Log Analytics Workspace**.
2. Select **Logs** from the left-hand menu.
3. You can run Kusto Query Language (KQL) queries. For example, to view Container App console logs:
   ```kusto
   ContainerAppConsoleLogs_CL
   | project TimeGenerated, ContainerAppName_s, Log_s
   | order by TimeGenerated desc
   ```
   To view Application Gateway access logs:
   ```kusto
   AzureDiagnostics
   | where ResourceProvider == "MICROSOFT.NETWORK" and Category == "ApplicationGatewayAccessLog"
   ```
